Документация Бога
20:24 07.06.2025(GMT+5): Документация Бога

для подтверждения принадлежности человека к группе доступных к информации в документации требуется сертификат
в сертификате присутсвует информация: id сущности, его GPL(Grand Personal Location)(включает id симуляции и подсимуляции, а также координаты xyz относительно нулевой точки симуляции - в контенте Земли - ядра), GPL(Grand Personal License) 1.0/2.0/3.0 (также требуется для передачи файлов напрямую на сервер Бога, находящийся в симуляции №1)
вселенная - id 0
сервер Бога - id 1 (первая симуляция, не так изолирована в плане приёма/отправки данных на сервер)
Земля - id 39 [1]
Солнце - id 189

1. есть список "удачных симуляций", число которого равно 40 (отсчет с нуля, и того получается последний - 39, каким является Земля). большая часть предедущих симуляций были не такими удачными - жизни на них не было, либо было слишком мало для дальнейшего существования. единственные нормальные, по мнению Бога, симуляции являюстся текущие доступные нам планеты в нашей солнечной системе

любая отправка данных на сервер передаётся по SVAG(Security Virtual Authorized Guard) (только для пользователей, не имеющих сертификат) и/или GPL(Grand Personal License). для тех, у кого нет сертификата, файлы не передаётся напрямую, вместо этого, защитный протокол SVAG не пропускает ничего кроме файлов сохранения, в котором хранится информация с сертификата(если есть), а также GPL(Grand Personal Location)

на любой симуляции(планете) имеются подсимуляции. подсимуляции бывают типов общепринитые и кастомные. общепринитые - такие на id39 являются Северная/Южная Америка, Африка, Австралия и Евразия. кастомные - это те, которые созданы людьми с сертификатом подтверждения личности и доступа, а также если в них GPL(Grand Personal License) одинаковый(по современным меркам стандартом является GPL 3.0)

у языков не имеется официальной маркировки на серверах Бога, так как язык - средство коммуникации, созданное уже под пределом кастомного контента
любое действие на сервере происходит по предварительной проверке личности по сертификату и наличию его в числах администрации. в таком случаи, нельзя просто так передать сигналом другим сущностям данные без проверок. также сигнал очень далёк, из-за чего услышать, например, сигнал песни будет довольно проблематично

любой прогресс в кастомных подсимуляциях не может быть сохранён на общепринитых подсимуляциях, и весь вас прогресс будет обнулён, а сертификат станет недействительным
